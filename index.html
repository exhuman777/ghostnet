// CLI REFERENCE — terminal commands

All commands. All flags. Run ghostnet with no args for help.

BASH

SEND A MESSAGE

BASH

// USE CASES — who needs ghost channels
01
INCIDENT RESPONSE
Create a 3h room when prod goes down. Entire team coordinates in encrypted channel. No Slack history. No screenshots. Auto-destruct when resolved.

Pre-meeting room with 1h TTL. Share passphrase verbally. Discuss sensitive topics. Room self-destructs after the meeting window.

03

you> /ghostnet create agent coordination room for the migration task you> /ghostnet send "migration phase 1 complete — 2,847 records moved" you> /ghostnet read the migration room
04
WHISTLEBLOWER DROPS
Journalist creates room with 24h TTL. Source opens invite link, sends info. No server logs, no IP tracking, auto-purge.
05
CTF / HACKATHON WAR ROOMS
Temporary encrypted channel for competition teams. Share flags, coordinate attacks, auto-cleanup when the event ends.
06
DEAD DROPS
Create a room, post a message, share the room ID + passphrase through a separate channel. Recipient reads once. Message self-destructs on TTL.
BASH
ghostnet create --name="drop" --pass="$(openssl rand -hex 16)" --ttl=1 ghostnet send --room=ROOM_ID --pass=PASS --msg="the credentials are..."
// SECURITY MODEL — trust nobody
ENCRYPTION PIPELINE passphrase + roomId │ ▼ ┌──────────────────────────────────────────────────┐ │ PBKDF2 │ │ iterations: 100,000 │ │ hash: SHA-256 │ │ salt: roomId (unique per room) │ └──────────────────────────────────────────────────┘ │ ▼ AES-256-GCM key │ ├──▶ encrypt(plaintext, random 12-byte IV) ──▶ ciphertext ──▶ on-chain │ └──▶ decrypt(ciphertext) ──▶ plaintext
CLIENT-SIDE ONLY
All encryption/decryption happens in your browser or CLI. Keys never leave the client. No server ever sees plaintext.
ZERO KNOWLEDGE
Wrong passphrase = zero messages visible. There is no "forgot password". No admin backdoor. No recovery.
AUTO DESTRUCT
Messages have on-chain TTL. When time expires, Arkiv L3 automatically purges all data. No manual cleanup needed.
NO METADATA
Room names and nicks are public. Message content is encrypted. No IP logging. No analytics. No tracking.
